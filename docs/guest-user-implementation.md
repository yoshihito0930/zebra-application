# ゲストユーザー機能 実装計画

## 概要

ユーザー登録なしでスタジオ予約システムを利用できるゲストユーザー機能の実装計画。
段階的アプローチで、フロントエンドのみの実装から開始し、徐々にバックエンド対応を追加する。

---

## 実装フェーズ

### **フェーズ1: 閲覧のみゲスト対応（実装中）**

**バックエンド影響: なし**

#### 実装内容（フロントエンドのみ）

1. **認証ガードの調整**
   - カレンダーページ (`/customer/calendar`) を認証不要にする
   - プラン・オプション閲覧を認証不要にする（既にAPI側が対応済み）

2. **UIの調整**
   - ゲストユーザー向けに「ログインして予約」ボタンを表示
   - カレンダーは閲覧可能、予約作成ボタンをクリックするとログイン/登録画面へ誘導

3. **レイアウトの調整**
   - CustomerLayoutをゲストモードに対応
   - ログイン前: 「ログイン」「新規登録」ボタンを表示
   - ログイン後: ユーザーメニューを表示

#### 利用可能な機能（ゲストユーザー）

- ✅ カレンダー閲覧（空き状況確認）
- ✅ プラン一覧閲覧
- ✅ オプション一覧閲覧
- ❌ 予約作成（ログインが必要）
- ❌ 予約一覧・詳細閲覧（ログインが必要）
- ❌ 問い合わせ（ログインが必要）

#### 実装ファイル

- `src/components/auth/ProtectedRoute.tsx` - 認証ガードの作成
- `src/components/layouts/CustomerLayout.tsx` - ゲストモード対応
- `src/App.tsx` - ルーティング設定の更新
- `src/pages/customer/CalendarPage.tsx` - ゲスト向けUI調整

### **フェーズ2: ゲスト予約対応（将来実装）**

**バックエンド影響: あり（API変更が必要）**

#### バックエンド変更内容

1. **API仕様変更**
   - `POST /reservations` を認証オプショナルに変更
   - ゲスト予約用の新しいリクエストフィールド追加:
     ```json
     {
       "is_guest": true,
       "guest_name": "山田太郎",
       "guest_email": "guest@example.com",
       "guest_phone": "090-1234-5678",
       "guest_company": "株式会社サンプル"
     }
     ```

2. **予約確認トークン発行**
   - ゲスト予約時に一意のトークンを生成
   - メールで確認リンクを送信: `https://studio-zebra.com/reservations/guest/{token}`

3. **新しいエンドポイント追加**
   - `GET /reservations/guest/{token}` - ゲスト予約詳細取得
   - `PATCH /reservations/guest/{token}/cancel` - ゲスト予約キャンセル
   - `PATCH /reservations/guest/{token}/promote` - 仮予約→本予約切り替え

4. **データモデル変更**
   - `reservations` テーブルに以下フィールド追加:
     - `is_guest` (boolean)
     - `guest_name` (string, optional)
     - `guest_email` (string, optional)
     - `guest_phone` (string, optional)
     - `guest_company` (string, optional)
     - `guest_token` (string, unique, optional) - 予約確認用トークン
   - `user_id` フィールドをオプショナルに変更

#### フロントエンド変更内容

1. **予約作成フォーム**
   - ゲストモードの場合、連絡先情報入力欄を表示
   - 「アカウント作成して予約」と「ゲストとして予約」の選択肢を提供

2. **予約確認ページ**
   - トークンベースの予約詳細ページを作成
   - `/reservations/guest/{token}` ルートを追加

3. **メール通知**
   - 予約完了メールに確認リンクを含める
   - キャンセル・変更用のリンクも含める

#### 利用可能な機能（ゲストユーザー）

- ✅ カレンダー閲覧
- ✅ プラン・オプション閲覧
- ✅ **予約作成**（メールアドレスで確認）
- ✅ **予約詳細閲覧**（トークンベース）
- ✅ **予約キャンセル**（トークンベース）
- ❌ 予約一覧閲覧（ゲストは個別トークンのみ）
- ❌ 問い合わせ（次フェーズで対応）

## 技術的検討事項

### セキュリティ

1. **トークン生成**
   - UUID v4またはランダムな文字列（32文字以上）
   - 推測不可能であること
   - 有効期限を設定（例: 予約日+30日）

2. **メール認証**
   - トークンベースのアクセス制御
   - メールアドレスの検証（確認メール送信）

3. **スパム対策**
   - reCAPTCHAの導入を検討
   - 同一IPアドレスからの連続予約制限

### データ管理

1. **ゲスト予約の扱い**
   - 後からアカウント作成時に既存ゲスト予約を紐付け可能にする
   - メールアドレスベースでマッチング

2. **データ保持期間**
   - ゲスト予約データも通常予約と同様に10年間保持
   - トークンの有効期限は予約完了後30日

### UX考慮点

1. **ゲスト vs 会員登録の誘導**
   - 初回訪問者にはゲスト予約を推奨
   - リピーター向けに会員登録のメリットを説明（予約履歴閲覧、簡単予約など）

2. **予約確認の利便性**
   - メールに含まれる確認リンクをわかりやすく
   - ブックマーク推奨の案内
