# フロントエンドAPI連携計画

## APIクライアント設計

### 共通設定
- ベースURL設定
- 認証トークン管理
- リクエスト/レスポンスインターセプター
- エラーハンドリング
- リトライメカニズム

### 認証API
- **ログイン**: `POST /auth/login`
  - リクエスト: `{ email, password }`
  - レスポンス: `{ token, user }`
  - 処理: 認証トークンの保存、ユーザーデータの状態管理への格納

- **ユーザー登録**: `POST /users`
  - リクエスト: `{ name, email, password, phone_number, company_name, address }`
  - レスポンス: `{ user, token }`
  - 処理: アカウント作成後、自動ログイン処理

### カレンダーAPI
- **カレンダー情報取得**: `GET /calendar`
  - パラメータ: `{ date }`
  - レスポンス: `{ availableDates, bookedTimeSlots, tentativeBookings }`
  - 処理: カレンダー表示用データ整形、空き状況の視覚化

### 予約API（ユーザー向け）
- **自分の予約一覧取得**: `GET /reservations`
  - パラメータ: `{ status, reservation_type }`
  - レスポンス: `{ reservations: [...] }`
  - 処理: 予約リスト表示、ステータスに応じた表示分け

- **予約作成**: `POST /reservations`
  - リクエスト: 予約タイプ及び必要情報
  - レスポンス: 作成された予約情報
  - 処理: フォーム送信、成功時確認画面表示

- **予約変更**: `PUT /reservations/{id}`
  - リクエスト: 変更内容
  - レスポンス: 更新された予約情報
  - 処理: 変更確認ダイアログ、成功時リスト更新

- **予約削除**: `DELETE /reservations/{id}`
  - レスポンス: 成功ステータス
  - 処理: 削除確認ダイアログ、成功時リスト更新

### 管理者向けAPI
- **全予約一覧取得**: `GET /admin/reservations`
  - パラメータ: `{ month, date, reservation_type, sort, page, limit }`
  - レスポンス: `{ reservations: [...], total, page, limit }`
  - 処理: 管理画面での予約一覧表示、ページネーション処理

- **予約詳細取得**: `GET /admin/reservations/{id}`
  - レスポンス: 予約の詳細情報
  - 処理: 管理者用詳細表示画面の表示

- **予約承認**: `PUT /admin/reservations/{id}/approve`
  - リクエスト: `{ status }`
  - レスポンス: 更新された予約情報
  - 処理: 承認ボタン処理、カレンダー反映

- **ユーザー情報取得**: `GET /admin/users/{id}`
  - レスポンス: ユーザーの詳細情報
  - 処理: ユーザー情報パネル表示

## エラーハンドリング戦略
- APIエラーコードに応じた処理分岐
- ユーザーフレンドリーなエラーメッセージ表示
- ネットワークエラー時の再試行機能
- セッション切れ時の自動リダイレクト

## データキャッシュ戦略
- キャッシュが有効なデータ種別の特定
  - カレンダーデータ（短期間）
  - ユーザーデータ（セッション中）
  - 静的設定データ（長期間）
- キャッシュ無効化タイミング
  - 予約作成/変更/削除後
  - ログアウト時
  - 明示的な更新リクエスト時

## リアルタイム更新戦略
- ポーリングによる定期更新
  - カレンダー表示時（30秒〜1分間隔）
  - 管理者ダッシュボード（10〜30秒間隔）
- WebSocketによるリアルタイム更新（将来拡張）
  - 予約状況のリアルタイム反映
  - 管理者への通知

## オフライン対応戦略
- オフライン時の読み取り専用モード
- 操作のキューイングと再接続時の同期
- オフライン状態のユーザー通知

## プログレッシブデータ取得
- 必要なデータのみを初期ロード
- スクロールや操作に応じた追加データ取得
- 先読みによるパフォーマンス最適化
